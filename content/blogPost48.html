Building out a data platform on the cloud

As a senior engineering developer consultant, I was tasked with building out a new cloud-based big data solution based on Azure, Databricks, Airflow and Power BI for reporting and to complete the migration from on-premise Oracle to this new platform. In this blog post, I will discuss the approach, plan, data migration, and the various challenges encountered in building out this new data platform.

The Approach

Before we started the project, we had to decide on an approach. We decided to use a combination of Azure, Databricks, Airflow, and Power BI for our data platform. We felt that Azure provided the flexibility and scalability needed to handle large amounts of data. We also chose Databricks because it is an open-source platform that integrates with the other tools we were using. Airflow was chosen to handle the data pipelines and orchestration, and Power BI was chosen for reporting and analytics.

Data Migration

To complete the migration from on-premise Oracle to the new platform, we had to move the data from the Oracle database to Azure. We wrote a custom script to export the data from Oracle and then used Azure Data Factory to move the data to Azure. We also used Azure Data Lake Storage to store the data. Once the data was in Azure, we used Databricks to transform the data and get it into the desired format.

Data Pipelines and Orchestration

To handle the data pipelines and orchestration, we used Airflow. Airflow allowed us to easily automate the data pipelines and set up orchestrations for the different data sources. We wrote custom scripts to load data from various sources into the data lake and then used Airflow to orchestrate the process. This allowed us to easily monitor and manage the data pipelines.

Benefits of the Platform

We found that the combination of Azure, Databricks, Airflow, and Power BI provided us with a highly scalable and flexible data platform. It allowed us to easily move data from the on-premise Oracle database to Azure, and then use Databricks and Airflow to transform and orchestrate the data. It also allowed us to easily set up data pipelines and orchestration. Finally, we were able to use Power BI to easily create reports and visualizations.

Overall, the new data platform we built on Azure, Databricks, Airflow and Power BI was a success. We were able to easily migrate the data from Oracle to Azure, and then use Databricks and Airflow for data transformation and orchestration. We were also able to use Power BI for reporting and analytics. This new data platform will provide us with the flexibility and scalability we need to handle large amounts of data.
